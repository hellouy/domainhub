# 多人协作 / 迁移指南 —— 共享同一个数据库

本项目的数据**不存在 Vercel 里**,而是存在独立的 **Neon Postgres** 数据库中。
Vercel 只保存一个指向它的连接串(环境变量 `DATABASE_URL`)。

> 结论:只要每个迭代者在自己的 Vercel 项目里填入**同一个 `DATABASE_URL`**,
> 大家连的就是同一份数据——**零迁移、无需导入导出、数据原封不动、实时同步**。

---

## 一、数据为什么"看起来会丢"

- 代码仓库(GitHub)里 **不包含** 数据库连接串和密钥(出于安全,不能提交)。
- 别人 clone / fork 代码后,环境变量是空的 → 连不上数据库 → 页面报错、看似"数据没了"。
- 其实数据一直在 Neon 里,丢的只是"连接凭证"。把凭证补上即可恢复。

---

## 二、新迭代者接入步骤(共享同一份数据)

### 1. 拿到代码
从 GitHub 仓库 clone 或 fork。

### 2. 索取环境变量
向主账户持有者索取以下变量的**真实值**。持有者可在
**Vercel 项目 → Settings → Environment Variables** 里逐个复制(点右侧眼睛图标查看):

| 变量 | 必填 | 说明 |
| --- | --- | --- |
| `DATABASE_URL` | ✅ | **共享的核心**,填相同值即连同一份数据 |
| `ADMIN_PASSWORD` | ✅ | 后台登录密码 |
| `EXCHANGERATE_API_KEY` | 建议 | 汇率主通道密钥 |
| `CURRENCYFREAKS_API_KEY` | 建议 | 汇率备用通道密钥 |
| `CREDENTIAL_ENCRYPTION_KEY` | 视情况 | 用了注册商凭据功能则必填,且须与主账户相同 |
| `CRON_SECRET` | 可选 | 定时爬虫接口鉴权 |
| `AI_GATEWAY_API_KEY` | 可选 | AI 自愈功能;v0/Vercel 环境常零配置 |

完整清单见仓库根目录的 `.env.example`。

### 3a. 本地开发
把值写入项目根目录的 `.env.local`(参照 `.env.example` 的格式),然后:

\`\`\`bash
npm install
npm run dev
\`\`\`

无需建表、无需 seed——因为库里已经有全部数据。

### 3b. 部署到自己的 Vercel
1. 在自己的 Vercel 里 New Project,导入这个仓库。
2. 在 **Environment Variables** 里把上面那些变量按相同值粘贴进去。
3. Deploy。完成后你的部署连的就是同一份数据。

---

## 三、几个重要提醒

- **共享 = 同一份数据**:任何人在后台增删改,所有部署会同时看到。适合团队协作,
  但要注意一个人误删会影响所有人。若某人想做破坏性实验,建议他单独开一个 Neon 库。
- **千万不要把真实密钥提交进 Git**。`.env.local` 已被 `.gitignore` 忽略;
  分发密钥请走私密渠道(如密码管理器、私聊),不要写进代码或公开文档。
- **`DATABASE_URL` 用带 `-pooler` 的连接串**(连接池版),更适合 serverless 部署。
- 想临时给某人一份"只读"数据副本,可在 Neon 控制台用 **Branch** 功能一键复制整库,
  再把分支连接串给他——这样他改动不影响主库。

---

## 四、如果将来想彻底独立(不共享)

在 Neon 控制台对当前库做一次 **Branch / 快照**,得到一份完全相同的数据副本,
把新分支的连接串填给对方即可。他从此拥有独立的一份相同数据,互不影响。
